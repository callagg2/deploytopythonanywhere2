Short answer: you almost always add a watermark image through CSS, not HTML or JavaScript.

Here’s the breakdown so you can choose the right approach depending on what you want.

🎯 Best practice: Add the watermark in your CSS
CSS is designed for layout, backgrounds, and visual effects — which is exactly what a watermark is.

Example (CSS background watermark)
css


Copy
body {
    background-image: url('watermark.png');
    background-repeat: no-repeat;
    background-position: center;
    background-size: 300px;
    opacity: 0.1; /* optional if you want transparency */
}
Or, if you want it behind only a specific container:

css


Copy
.container {
    background-image: url('watermark.png');
    background-repeat: no-repeat;
    background-position: center;
    background-size: contain;
    opacity: 0.1;
}
This keeps your HTML clean and avoids unnecessary JavaScript.

🧩 When to put it in HTML
Use HTML only if the watermark is an actual visible element, not a background.

Example:

html


Copy
<img src="watermark.png" class="watermark">
Then style it with CSS:

css


Copy
.watermark {
    position: absolute;
    top: 50%;
    left: 50%;
    width: 300px;
    opacity: 0.1;
    transform: translate(-50%, -50%);
    pointer-events: none;
}
⚙️ When to use JavaScript
Use JS only if the watermark changes dynamically, such as:

User uploads an image and you overlay a watermark

You generate a canvas image with a watermark

You want to toggle the watermark on/off

Example (JS adding watermark dynamically):

js


Copy
const watermark = document.createElement("img");
watermark.src = "watermark.png";
watermark.className = "watermark";
document.body.appendChild(watermark);
🧠 For your API front end
Since your front end is static HTML/CSS/JS calling an API:

If the watermark is part of the page design → CSS

If the watermark overlays API data (like book covers) → JS

If the watermark is decorative and always visible → CSS

Given your CRUD app and Bootstrap layout, the cleanest solution is:

Add the watermark in your CSS file.
It keeps your HTML simple and avoids mixing logic with visuals.

function getOneBookItem() {

  fetch(`${url}/${id}`)
    .then(response => {
      if (!response.ok) {
        throw new Error("Book not found");
      }
      return response.json();

Your table header and table body now have different widths because of one specific change in your CSS:

❗ You increased the table title <h2> font size to 44px
From your CSS:

css


Copy

  margin: 5px 0 0;
  font-size: 44px;
}
A 44px heading is huge — it forces the header row to expand vertically and horizontally, which breaks alignment with the table below.

That’s why your header no longer matches the table width.

Let’s fix it cleanly.

✅ Fix 1 — Reduce the table title font size
Use something more reasonable, like:

css


Copy

  margin: 5px 0 0;
  font-size: 24px;   /* original size */
}
Or if you want it slightly larger:

css


Copy

This alone will realign the header and table widths.

✅ Fix 2 — Remove the negative margins you added
You currently have:

css


Copy

  margin: -10px -10px 0px 0px;
}
This makes the header wider than the table wrapper.

Replace it with:

css


Copy
.table-title {
  margin: 0 0 10px 0;
}
This ensures the header sits inside the wrapper, matching the table width exactly.

✅ Fix 3 — Ensure consistent padding
